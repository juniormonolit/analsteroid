# Design Guidelines — Монолитика

> Обязательные к соблюдению правила визуального языка приложения. Пополняется по мере
> появления новых стандартизированных решений (см. `WORKLOG.md` за датой правки).

---

## Анимации (правило, введено 09.07.2026)

**Требование владельца: «все будущие анимации делать именно так» — обязательно для
ЛЮБОЙ новой анимации в приложении, без исключений.**

### Стандарт

- **Длительность**: 250–300мс (эталон — `280ms`). Раньше в проекте использовались
  разные значения ~150–180мс — визуально ощущались резко/«дёргано». Дольше 300мс —
  уже ощущается медленно и тормозит работу с интерфейсом, короче 250мс — не успевает
  прочитаться как плавное движение.
- **Кривая**: `cubic-bezier(0.32, 0.72, 0, 1)` — классический iOS/macOS деселерейт
  (быстрый разгон, мягкое торможение к концу). Не использовать линейные (`linear`) или
  стандартные CSS (`ease`, `ease-in-out`) кривые для панелей/шторок/значимых переходов.
- **Переменные**: обе величины заданы в `app/globals.css` как CSS custom properties в
  `:root`:
  ```css
  --anim-duration: 280ms;
  --anim-ease: cubic-bezier(0.32, 0.72, 0, 1);
  ```
  Новая анимация обязана ссылаться на `var(--anim-duration)` / `var(--anim-ease)` —
  **не хардкодить** свои мс/кривые в компоненте или отдельном keyframe-классе. Если
  нужно поменять общий тайминг приложения — правка в одном месте (`globals.css`)
  применяется ко всем анимациям сразу.
- **Исключение — ховеры**: наведение курсора (hover-состояния кнопок/ссылок/строк
  таблицы, `.hover-reveal`) должно оставаться быстрым (~150мс, значение по умолчанию
  `transition-colors`/`duration-150` из Tailwind) — на них правило НЕ распространяется.
  Замедление отклика на hover ощущается как «тормозит интерфейс», а не как приятная
  анимация.
- **Готовые классы** (использовать вместо новых самописных):
  - `.slide-panel-in-right` / `.slide-panel-out-right` / `.slide-panel-in-left` /
    `.slide-panel-out-left` — выезд/выезд обратно правых и левых панелей.
  - `.slide-backdrop-fade` — плавное появление/скрытие затемняющей подложки под
    панелью (используется в компоненте `<SlideBackdrop>`, `components/ui/SlideBackdrop.tsx`,
    и подключается напрямую там, где вёрстка подложки нестандартна — см. комментарии
    в файле).
  - `.metric-compare-col-enter` / `.metric-compare-col-exit` — плавное появление/
    схлопывание колонок сравнения в `ReportTable.tsx`.
- **JS-таймеры**: если анимация управляется и CSS, и JS-таймером (например,
  `COMPARE_CLOSE_MS` в `features/reports/ui/ReportTable.tsx`, который держит колонки
  смонтированными на время fade-out), JS-константа должна численно совпадать со
  значением `--anim-duration` (JS не умеет читать CSS-переменные напрямую) — при
  изменении длительности в `globals.css` обязательно поправить и такие константы.
- **`prefers-reduced-motion: reduce`**: анимации должны отключаться (см. media-query
  в конце `globals.css`) — это правило не отменяется новым стандартом.

### Как добавить новую анимацию

1. Не писать `animation: my-thing 150ms ease-out` — использовать
   `animation: my-thing var(--anim-duration) var(--anim-ease)`.
2. Для CSS-переходов (`transition`) — `transition: <property> var(--anim-duration) var(--anim-ease)`,
   не `duration-150`/`duration-200`/произвольные мс через Tailwind-утилиты.
3. Если это hover/фокус-отклик — можно и нужно оставить быстрым (не подключать эти
   переменные).
